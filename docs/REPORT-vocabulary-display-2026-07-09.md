# 报告:词汇表在假设区怎么显示

- 日期:2026-07-09
- 作者:小红
- 背景:方案 B 把每关词库当"已知条件"发给玩家,面板上会出现一排词汇假设。
  怎么显示这一排,是纯教学/审美决定。本报告穷举可行方案、给出取舍。

## 1. 问题

一个词库项要同时呈现两样东西:**词**(surface form)和**范畴**(N/D/V…)。
它在 Lean 里是一条假设,面板固定长成:

```
<名字> : <类型>
```

- **名字**(冒号左)= 假设的 fvar 名,由 lean4game 直接渲染,我**改不了它的渲染规则**。
- **类型**(冒号右)= 我用 delaborator **完全控制**,可渲染成任意语法。

词和范畴各放哪、用什么符号,就是全部的设计自由度。

### 评判标准

| 标准 | 含义 |
|---|---|
| 统一 | 单词 / 多词 / 外语,看起来一个样 |
| 扛外语 | "Puerto Rico"、"猫"、"naïve" 都不崩、不变丑 |
| 无冗余 | 词只出现一次 |
| 符号干净 | 不带 `«»` 书名号、不带多余括号、不像中文全角 |
| 左侧无噪音 | 名字不是没意义的占位符 |

终局是"玩家给**外语**造语法"(含 CJK、带空格的专名),所以**扛外语**权重最高。

## 2. 死约束(已实测,不是猜)

名字(冒号左)这一侧,Lean 的渲染规则是硬的:

| 名字来源 | 显示成 | 结论 |
|---|---|---|
| `cat` / `will` / `by`(合法标识符,含关键字) | `cat` `will` `by` | 裸,干净 |
| `niño` / `naïve`(带重音的拉丁字母) | `niño` `naïve` | 裸,干净 |
| `猫`(CJK) | `«猫»` | **书名号回来** |
| `Puerto Rico`(带空格) | `«Puerto Rico»` | **书名号回来** |
| 源码写 `«cat»` 想强制书名号 | `cat` | **抹掉**,逼不出统一书名号 |
| 名字里塞引号 `«"cat"»` | `«"cat"»` | 书名号 + 引号,更糟 |

冒号右(类型)这一侧,我能渲染成任意样子,但两个符号有坑:

- **ASCII 冒号 `:`** 不能自定义当分隔符——它是 Lean 类型标注的保留符号,一注册就
  把满屏别处的 `x : T` 全搞崩(实测)。只能借用内置类型标注,那样**带一对括号**:`("cat" : N)`。
- **破折号 `—`、方括号 `[]`** 不是保留符号,可干净当分隔符,**不带括号**。

**关键推论**:词若坐在**名字**位置,CJK 和多词一定套书名号 → 天生不统一、且正好
在外语处翻车。词若作**带引号的字符串**放在**类型**位置,引号是统一加的、字符串什么
都吃得下 → 统一且扛外语。**所以词想干净,就该待在冒号右边。**

## 3. 可行路线

### 甲 · 词当名字,右边只放范畴
```
cat : N
niño : N
«猫» : N            ← CJK 翻车
«Puerto Rico» : N   ← 多词翻车
```
最省,词只出一次,用 Lean 自带冒号。但 CJK/多词套书名号,**不统一、外语常翻车**。

### 乙 · 词作引号字符串放右边,名字当中性把手
分隔符可选(都不带括号):
```
w₁ : "cat" — N            w₁ : "cat" [N]            (破折号 / 方括号)
w₂ : "Puerto Rico" — N    w₂ : "Puerto Rico" [N]
w₃ : "猫" — N             w₃ : "猫" [N]
```
**统一、扛任何语言**。代价:左边 `w₁` 是没意义的把手。
(`[N]` 这个写法对句法学家其实很亲切——就是 labeled bracketing,和本项目 `plot`
输出 `[ɴᴘ …]` 一脉相承。)

### 丙 · 词两边都放(NNG 式:名字=把手,类型=完整事实)
```
cat : "cat" — N
«Puerto Rico» : "Puerto Rico" — N
```
其实这正是 NNG 的常态:`add_comm : a + b = b + a`,名字 + 完整陈述并列,没人嫌冗余。
右边永远完整干净;左边是可引用的把手(将来能支持 `head cat` 直接点名)。
代价:词出现两次;左边多词/CJK 仍套书名号。

### 丁 · 词作字符串放右边,名字**藏起来**(待验证,可能最漂亮)
理想是面板只显示一排**没有名字**的类型:
```
"cat" — N
"Puerto Rico" — N
"猫" — N
```
= 一张纯净的"词 — 范畴"词卡表,零 Lean 噪音、零冗余、统一、扛外语。**这是最理想的。**
悬念:需要 lean4game 能"显示一条假设的类型、但藏掉它的名字"。inaccessible 名字在
普通 Lean 里显示成 `x✝`(带匕首),不是真的藏掉;lean4game 面板会不会真藏、或有没有
专门注解,**只有实机能验**(地雷 5/8:本地起不了前端)。

## 4. 对照

| | 统一 | 扛外语 | 无冗余 | 符号干净 | 左侧无噪音 |
|---|:--:|:--:|:--:|:--:|:--:|
| 甲 词=名字 | ✗ | △ | ✓ | 拉丁✓/外语✗ | ✓ |
| 乙 词=右串+中性名 | ✓ | ✓ | ✓ | ✓ | ✗ |
| 丙 词两放(NNG式) | △ | ✓ | ✗ | 右✓/左△ | △ |
| 丁 词=右串+藏名 | ✓ | ✓ | ✓ | ✓ | ✓✓ |

## 5. 建议

1. **先花五分钟验丁**:实机发一条名字 inaccessible 的词库假设,看 lean4game 面板到底
   把它显示成"纯类型一行"还是"`x✝ : 类型`"。若能纯类型 → **丁 胜出**,一张无名词卡表,
   最漂亮。这是唯一能同时满足全部标准的方案,值得先探。
2. **丁 不成 → 走 乙**:统一、扛外语最稳,只是左边一个中性把手。把手可用 `w₁ w₂…`,
   或干脆等真上外语再纠结。分隔符我偏 `[N]`(句法学家的母语)或 `—`(你选的破折号)。
3. **甲** 只在"永远只有英文单词"时才够用——和外语终局冲突,不建议做主选。

一句话:词想干净必须待在冒号右边当引号字符串(乙/丁);差别只在左边那个名字能不能藏
(丁)或只能当中性把手(乙)。**丁 若实机可行就是答案,否则乙。**
