# What Was Done

## Engine (XSyntax/Tactics.lean)

- `isModal` + overt-T support in `splitHeadComplementTarget`: a T-selects-V
  complement now puts a leading modal (will/can/may/must/shall/would/could/
  might/should/did/does) at T⁰, else falls back to a null T head (unchanged
  for present-tense levels and the boss).
- `isClauseBoundaryWord` in `splitSpecifierTarget`: subject-boundary scan now
  stops at a modal too, not just the predicate whitelist — so "cats will sleep"
  splits subject "cats" / T′ "will sleep" correctly.
- New 8th player command `cannotSelect` (elab): proves `¬ Selects c d` by
  exhausting the licensing table's constructors. Refuses in linguistics if the
  pair is actually licensed (checks `isLicensedPair` BEFORE `cases`, so a
  licensed pair yields the linguistic message, not Lean's "unsolved goals").
  Note: named `cannotSelect` (single camelCase token) rather than the
  maintainer-suggested two-word "cannot select" — two-word tactic tokens were
  not verified feasible and the project has prior token-collision landmines;
  can revisit if a two-word player-facing spelling is still wanted.
